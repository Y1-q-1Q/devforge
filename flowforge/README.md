# FlowForge - 工作流自动化

> 让重复工作自动化，提升开发效率

---

## 🎯 产品定位

**一句话**: 为开发者打造的自动化工作流引擎

**核心价值**:
- 自动化重复任务
- 连接不同工具
- 可视化流程编排
- 节省开发时间

---

## 🏗️ 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                     🎨 可视化编辑器                           │
│   拖拽节点 → 连接流程 → 配置参数 → 运行测试                   │
└───────────────────────┬─────────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────────┐
│                     ⚙️ 工作流引擎                            │
│   流程解析 → 任务调度 → 执行器 → 状态管理                     │
└───────────────────────┬─────────────────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────────────────┐
│                     🔌 连接器生态                            │
│   GitHub / GitLab / Slack / 邮件 / Webhook / API / ...       │
└─────────────────────────────────────────────────────────────┘
```

## 🛠️ 技术栈

### 前端
- React + TypeScript
- React Flow (可视化编排)
- TailwindCSS

### 后端
- Node.js + Express
- Bull (任务队列)
- Redis (状态存储)
- SQLite (流程定义)

### 执行引擎
- vm2 (沙箱执行)
- child_process (Shell执行)
- axios (HTTP请求)

## 💡 典型使用场景

### 场景1: 自动化部署
```
Git Push → 运行测试 → 构建 → 部署 → 通知
```

### 场景2: 定时任务
```
每天9点 → 拉取数据 → 处理 → 生成报告 → 发送邮件
```

### 场景3: 代码审查
```
PR创建 → 自动Review → 检查规范 → 评论反馈
```

### 场景4: 数据同步
```
数据库变更 → 转换格式 → 同步到ES → 更新缓存
```
