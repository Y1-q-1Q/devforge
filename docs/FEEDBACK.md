# 📣 用户反馈收集

## 如何提交反馈

### 方式1: GitHub Issues (推荐)
https://github.com/Y1-q-1Q/devforge/issues

**提交模板**:
```
## 反馈类型
- [ ] Bug报告
- [ ] 功能建议
- [ ] 使用体验
- [ ] 其他

## 描述
详细描述你的反馈...

## 使用场景
你在什么情况下遇到这个问题/需要这个功能？

## 期望
你希望得到什么结果？

## 环境
- 操作系统: 
- 浏览器/VSCode版本: 
- DevForge版本: 
```

## 反馈分类

### 🐛 Bug报告
- 功能不正常
- 界面显示问题
- 性能问题

### 💡 功能建议
- 新功能需求
- 现有功能改进
- 集成建议

### 💬 使用体验
- 交互反馈
- 界面设计
- 文档问题

### 🎯 其他
- 任何想法

---

## 反馈处理流程

1. **收集** - 24小时内确认收到
2. **分类** - 标记优先级
3. **讨论** - 社区讨论可行性
4. **实现** - 开发迭代
5. **发布** - 更新版本

---

## 反馈奖励

- 被采纳的反馈者会出现在 CONTRIBUTORS.md
- 重大贡献者获得项目维护权限
