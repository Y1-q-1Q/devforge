## 模块职责

### KnowForge (知识管理)
**职责**: 知识采集、存储、检索、输出
**技术栈**: Node.js + Express + JSON存储
**用户**: 所有开发者

### CodeForge (AI编程助手)
**职责**: 代码生成、解释、审查、重构
**技术栈**: TypeScript + VSCode Extension API + OpenAI
**用户**: VSCode用户

### FlowForge (规划中)
**职责**: 工作流自动化、任务编排
**技术栈**: 待定
**用户**: 需要自动化的开发者

### DeployForge (规划中)
**职责**: 一键部署、CI/CD
**技术栈**: 待定
**用户**: 需要快速部署的开发者

## 技术选型理由

### 为什么用Node.js？
- 前后端统一语言
- 丰富的npm生态
- 快速开发

### 为什么用TypeScript？
- 类型安全
- 更好的IDE支持
- 易于维护

### 为什么用JSON存储？
- 开发阶段简单快速
- 易于迁移到数据库
- 无依赖
